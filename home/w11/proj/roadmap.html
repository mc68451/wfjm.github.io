Short term (in work)

  • proper integration of CPU monitoring unit (dmcmon) and hardware break point unit (dmhbpt) with w11 core and back end software.
  • lots of verification topics

Mid term

  • DEUNA Ethernet; 2.11BSD on w11a on the internet.
  • DZ11 terminal multiplexer; classical multiuser operation.
  • fifo buffered implementation of character devices (DL11,PC11,LP11).
  • KW11P: programmable clock (needed for 2.11BSD kernel profiling).
  • CPU throttling mechanism (some old code, especially tests with timing loops, has problems on a fast machine...).
  • port to ZTEX USB-FPGA Module 2.13. The ZTEX 2.13 combines Atrix-7 and Cypress FX2, and has a fast JTAG over USB, and is therefore a good candidate for 'yet another port'.

Long term

  • the current w11a system will be frozen when the above mentions I/O system components are implemented.
  • w11f: a w11a plus a FPP Floating Point Processor. The FPP will be asynchronous, with a more efficient coupling than in the original KB11-C CPU plus FP11-C FPP combination.
  • w11m: a w11f with a new cache system, now write-back, wider cache lines, suitable to build a 11/70MP multiprocessor system. In contrast to the original 11/70MP a state-of-the-art cache coherence will be implemented.
  • w11e:a stripped down w11a optimized as an embedded console and IO processor in larger systems.
  • w11n: a w11m with a w11e as CIOP (console and IO processor) which can be operated stand-alone without a backend server.
  • w11p: new CPU with a much improved micro architecture. With micro- and macro-pipelining and a significantly improved CPI, better than the J11. Much faster than the w11a and derivatives, but also significantly higher resource consumption. Will implement, finally, the 11/44 and J11 special instructions (MFPT, MTPS, MFPS, TSTSET, WRTLCK), and make the processor type selectable.

Long term goal is to re-create a PDP-11/70MP (aka 11/74) system. This up to 4 CPU symmetric multiprocessor system was developed in the late '70's by DEC, but was never marketed. See article {% include wbm-link.html date="20160529061435" link="http://www.village.org/pdp11/faq.pages/never11s.html" text="The Never 11's" %}. A few prototypes were build, see Figure R-2 for a picture of PHEANX/CASTOR. Some documentation has survived, like the preliminary DEC 11/70MP system manual, see also a historical summary. Full SMP support was added around 1980 to the RSX-11M-PLUS V2.0 operating system, see for example the set and remove affinity system calls in the Executive Manual (STAF$ on p 5-193 and RMAF$ on p 5-128). Worth to note in this context is that already in 1972 a 16 node SMP system based on PDP-11 CPU's was build by W. Wulf and G. Bell at Carnegie Mellon, the C.mmp, see their paper "C.mmp--A Multi-mini-processor" for the Fall JCC 1972.
